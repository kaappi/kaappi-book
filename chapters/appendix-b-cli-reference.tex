\chapter{Kaappi CLI Reference}
\index{CLI reference}\index{command line}
\label{app:cli}

\section{Synopsis}

\begin{Verbatim}[fontsize=\small]
kaappi [OPTIONS] [FILE]
kaappi compile FILE [-o OUTPUT]
\end{Verbatim}

With no arguments, launches the REPL. With a file argument, executes the Scheme source file. Input can also be piped: \texttt{echo '(+ 1 2)' | kaappi}.

\section{Library Search Order}

When resolving \texttt{(import (lib name))}, Kaappi searches for \texttt{lib/name.sld} in:

\begin{enumerate}
    \item Current working directory (\texttt{./})
    \item The \texttt{./lib/} subdirectory
    \item Directories from \texttt{--lib-path} flags (in order)
    \item \texttt{\textasciitilde/.kaappi/lib/} (user library directory, where thottam installs packages)
\end{enumerate}

\section{The REPL}
\index{REPL}

The REPL prompt is \texttt{kaappi>}. Expressions with unbalanced parentheses automatically continue on the next line. History is saved to \texttt{\textasciitilde/.kaappi/history} (up to 1000 entries). Exit with \texttt{,quit}, \texttt{(exit)}, or Ctrl-D on an empty line.

\texttt{,time expr} & Measure execution time \\

\texttt{,breakpoints} & List active breakpoints \\

\section{Thottam Package Manager}
\index{thottam}

\begin{Verbatim}[fontsize=\small]
thottam COMMAND [ARGS]
\end{Verbatim}

\begin{center}
\small
\begin{tabularx}{\textwidth}{@{}lY@{}}
\textbf{Command} & \textbf{Description} \\
\hline
\texttt{thottam install PKG} & Install a package and its dependencies \\
\texttt{thottam remove PKG} & Remove an installed package \\
\texttt{thottam list} & List all installed packages \\
\texttt{thottam update} & Update all packages to latest versions \\
\texttt{thottam update PKG} & Update a specific package \\
\texttt{thottam verify} & Check installed packages against the lockfile \\
\texttt{thottam --locked install PKG} & Refuse to install packages not in the lockfile \\
\texttt{thottam --version} & Show thottam version \\
\texttt{thottam --completions SHELL} & Output shell completions (bash, zsh, fish) \\
\end{tabularx}
\end{center}

\texttt{install} accepts an optional version pin and source URL:

\begin{Verbatim}[fontsize=\small]
thottam install PKG[@VER][::URL]

thottam install kaappi-web              # latest release
thottam install kaappi-web@v1.0.0       # exact tag
thottam install kaappi-net@">=0.2.0"    # version range
thottam install kaappi-auth::https://github.com/bob/kaappi-auth
\end{Verbatim}

Packages are installed to \texttt{\textasciitilde/.kaappi/lib/}, and the lockfile is kept at \nolinkurl{~/.kaappi/lock.json}.

\section{Native Compilation}

Compile a Scheme program to a standalone native binary:

\begin{Verbatim}[fontsize=\small]
kaappi compile program.scm -o myapp
./myapp
\end{Verbatim}

Requirements:
\begin{itemize}
    \item LLVM must be available on the system (the compiler emits LLVM IR and invokes \texttt{clang})
    \item \texttt{libkaappi\_rt.a} must be built (\texttt{zig build lib})
\end{itemize}

To inspect the generated LLVM IR without compiling:

\begin{Verbatim}[fontsize=\small]
kaappi --emit-llvm program.scm -o program.ll
\end{Verbatim}

\section{Bytecode Caching}

When Kaappi runs a file, it caches the compiled bytecode in a \texttt{.sbc} file alongside the source. On subsequent runs, if the source has not changed (by timestamp), the cached bytecode is loaded directly---skipping the reader, expander, and compiler passes.

To force recompilation, delete the \texttt{.sbc} file or modify the source.

\section{Sandbox Mode}
\label{sec:sandbox-mode}

\texttt{--sandbox} restricts what a program can do:

\begin{itemize}
    \item FFI is disabled (\texttt{ffi-open}, \texttt{ffi-fn}, and the other \texttt{ffi-*} procedures raise errors)
    \item File I/O is disabled (\texttt{open-input-file}, \texttt{call-with-output-file}, \texttt{file-exists?}, \texttt{delete-file}, and the rest of \code{(scheme file)})
    \item All SRFI-170 filesystem operations are disabled
    \item \texttt{eval} and \texttt{load} are disabled
    \item Process context is blocked: \texttt{get-environment-variable}, \texttt{command-line}, \texttt{exit}
    \item OS threads (SRFI-18) are disabled
\end{itemize}

Pure computation, standard output (\texttt{display}, \texttt{write}), and green fibers remain available.

Useful for running untrusted code safely:

\begin{Verbatim}[fontsize=\small]
kaappi --sandbox --timeout 5000 --max-memory 50000000 untrusted.scm
\end{Verbatim}
